# Fluent Self — план: от клона к своему сайту

> Сейчас: pixel-perfect основа firstuk + слой modernize (config, формы, SEO).
> Цель: **свой** сайт Fluent Self, не копия чужого бренда.

---

## Принцип работы

**Не переписываем всё сразу.** Идём слоями — каждый этап даёт рабочий сайт, который можно показать заказчику.

```
Этап 0 ✅  Клон + техническая база (сейчас)
    ↓
Этап 1     Контент и бренд (ваши тексты, фото, контакты)
    ↓
Этап 2     Структура под Fluent Self (убрать лишнее, связать с kids)
    ↓
Этап 3     Рефакторинг кода (partials, content.js — как kids-сайт)
    ↓
Этап 4     Свой визуал (пастель, если нужно — без полного redesign)
```

---

## Этап 1 — Контент и бренд (1–2 итерации)

**От заказчика собрать:**

| Блок | Что нужно |
|------|-----------|
| Контакты | Телефон, адрес, WhatsApp, карта, юрлицо |
| Языки | Какие реально преподают + цены/форматы |
| Фото | Интерьер, занятия, 1 логотип |
| Тексты | О школе, преимущества своими словами |
| Отзывы | 3–5 реальных (без First UK / Залины) |
| Соцсети | VK, Telegram, Instagram — или скрыть |

**В коде:**

- Заполнить `js/config.js` на сервере
- Заменить фото: скачать в `assets/media/`, обновить пути в HTML
- Переписать отзывы на главной
- Убрать или заменить блок «известные ученики» → «Наши студенты» / кейсы / цифры

**Результат:** сайт уже **ваш по смыслу**, дизайн ещё как firstuk.

## Этап 3 — Рефакторинг (как Fluent Self kids)

Сейчас 9 HTML × ~900 строк — дубли header/footer. Цель:

```
partials/header.html, footer.html
js/layout.js          — подгрузка partials
js/content.js         — курсы, отзывы, события (данные, не HTML)
scripts/build-site.js — только если нужен импорт из референса
```

**Порядок:**

1. Вынести header/footer
2. Перенести отзывы и список курсов в `content.js`
3. Генерировать карточки курсов из данных
4. Удалить `_ref-*.html` когда референс больше не нужен

**Результат:** правки в одном месте, не в 9 файлах.

---

## Этап 4 — Свой визуал (опционально)

Если заказчик хочет **пастель / спокойнее**, чем чёрно-белый firstuk:

- Не трогать `index.min.css` — добавить `css/brand.css` с CSS-переменными
- Постепенно: фон, акцент `#4b7d72` → мятный, карточки курсов
- Свой hero вместо TypeIt (или свои фразы в JS bundle)

Делать **после** контента — иначе два раза переделываем.

## Как работать в git

| Ветка | Назначение |
|-------|------------|
| `master` | прод, то что на Reg.ru |
| `content` | тексты и фото от заказчика |
| `refactor` | partials, content.js |

**На сервере:** только `git pull` + свой `js/config.js` (не в git).
